# Проверка настроек DigitalOcean Spaces

## Шаг 1: Проверьте права доступа к Space

1. Откройте https://cloud.digitalocean.com/spaces/su-library
2. Перейдите в **Settings**
3. В разделе **File Listing** убедитесь, что выбрано **Public** (не Restrict File Listing)
4. Нажмите **Save**

## Шаг 2: Проверьте ключи доступа

1. Перейдите на https://cloud.digitalocean.com/account/api/spaces
2. Найдите ваш ключ: **key-1762493805669**
3. Убедитесь, что он активен и имеет права на запись

### Возможно нужно создать новый ключ с правами на запись:
1. Нажмите **Generate New Key**
2. Дайте имя: `e-library-write`
3. Сохраните новые Access Key и Secret Key
4. Обновите `.env` файл

## Шаг 3: Проверьте ACL настройки

Возможно Space настроен только на чтение. Нужно включить публичный доступ.

## Временное решение

Попробуйте использовать локальное хранилище для разработки:

В файле `.env` измените:
```
USE_SPACES=False
```

Это позволит загружать файлы локально пока мы настраиваем Spaces.
